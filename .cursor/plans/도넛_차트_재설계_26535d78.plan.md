# 도넛 차트 세그먼트 재설계

## 변경 목표


| 구분          | 기존      | 변경 후                   |
| ----------- | ------- | ---------------------- |
| 일간 도넛 세그먼트  | 미완료 1개  | 진행중 / 기한초과 / 완료 3개     |
| 일간 도넛 표시 조건 | 미완료 > 0 | 미완료(진행중+기한초과) > 0      |
| 일간 칭찬 릴레이   | 미완료 = 0 | **유지** — 일정 있고 미완료 = 0 |
| 주간 도넛 세그먼트  | 미완료 1개  | 완료 / 미완료 2개            |
| 주간 도넛 표시 조건 | 미완료 > 0 | 일정 있으면 항상 표시           |
| 주간 칭찬 릴레이   | 미완료 = 0 | **제거** — 완료 세그먼트로 대체   |
| 일정 없음 안내    | 유지      | 유지                     |


## 세그먼트 정의

**일간 (선택일 기준, `taskMatchesSelectedDay` 필터 적용)**

- `in-progress` 진행중: incomplete + (deadline 없음 OR deadline >= 선택일)
- `overdue` 기한초과: incomplete + deadline < 선택일
- `completed` 완료: completed 업무

**주간 (선택일이 속한 주, `taskOverlapsWeek` 필터 적용)**

- `completed` 완료: completed 업무
- `incomplete` 미완료: incomplete 업무

## 변경 파일

### 1. `[src/utils/task-feedback.ts](src/utils/task-feedback.ts)`

`DailyFeedbackResult` 인터페이스 교체:

```ts
export interface DailyFeedbackResult {
  inProgressOnDay: Task[]; // 진행중
  overdueOnDay: Task[]; // 기한초과
  completedOnDay: Task[]; // 완료
}
```

`WeeklyFeedbackResult` 인터페이스 교체:

```ts
export interface WeeklyFeedbackResult {
  completedInWeek: Task[];
  incompleteInWeek: Task[];
}
```

`computeDailyFeedback` - matched tasks를 3분류로 나눔:

- `deadline < dateKey` → overdueOnDay
- `deadline >= dateKey` 또는 없음 → inProgressOnDay
- completed → completedOnDay

`computeWeeklyFeedback` - `taskOverlapsWeek` 대상을 completed/incomplete로 분리

`shouldShowDailyPraise`, `shouldShowWeeklyPraise` - 기존 유지 (내부 로직은 incompleteOnDay 기준)

### 2. `[src/components/TaskPlanningPanel.vue](src/components/TaskPlanningPanel.vue)`

**색상 상수 추가:**

```ts
const COLOR_INPROGRESS = "#3b82f6"; // 파랑
const COLOR_OVERDUE = "#ef4444"; // 빨강
const COLOR_COMPLETED = "#22c55e"; // 초록
const COLOR_INCOMPLETE = "#f59e0b"; // 주황 (주간 미완료)
```

`**dailyDonutSegments` → 3개 세그먼트:

```ts
[
  {
    key: "in-progress",
    label: "진행중",
    count: inProgressOnDay.length,
    color: COLOR_INPROGRESS,
  },
  {
    key: "overdue",
    label: "기한초과",
    count: overdueOnDay.length,
    color: COLOR_OVERDUE,
  },
  {
    key: "completed",
    label: "완료",
    count: completedOnDay.length,
    color: COLOR_COMPLETED,
  },
];
```

`**weeklyDonutSegments` → 2개 세그먼트:

```ts
[
  {
    key: "completed",
    label: "완료",
    count: completedInWeek.length,
    color: COLOR_COMPLETED,
  },
  {
    key: "incomplete",
    label: "미완료",
    count: incompleteInWeek.length,
    color: COLOR_INCOMPLETE,
  },
];
```

**일간 표시 조건 (3단계 분기):**

```
hasDailyScheduledTasks
  ├─ 미완료(진행중+기한초과) > 0  →  도넛 (3세그먼트)
  ├─ 미완료 = 0, 완료 > 0        →  칭찬 릴레이 (기존 유지)
  └─ (fallback)                  →  "표시할 피드백이 없어요."
hasDailyScheduledTasks = false   →  "이 날짜에 배정된 일정이 없어요."
```

- `dailyUseDonut`: `inProgressOnDay.length > 0 || overdueOnDay.length > 0`
- `dailyShowRelay`: `hasDailyScheduledTasks && dailyUseDonut === false` (기존 로직 동일)

**주간 표시 조건 (2단계 분기):**

```
hasWeeklyScheduledTasks
  └─ 항상 도넛 표시 (완료 0이어도 표시, 완료 세그먼트가 칭찬 역할)
hasWeeklyScheduledTasks = false  →  "이번 주에 배정된 일정이 없어요."
```

- `weeklyUseDonut`: `hasWeeklyScheduledTasks`
- `weeklyShowRelay` 및 주간 릴레이 블록 **제거**

**상세 패널 (`task-planning__feedback-detail`) 업데이트:**

- daily `in-progress` → `inProgressOnDay` 목록
- daily `overdue` → `overdueOnDay` 목록
- daily `completed` → `completedOnDay` 목록
- weekly `completed` → `completedInWeek` 목록
- weekly `incomplete` → `incompleteInWeek` 목록

`**watch` 블록: 초기 선택 세그먼트를 새 key 기준으로 조정 (첫 번째 count > 0인 세그먼트 자동 선택)

### 3. 칭찬 릴레이 정리

**유지 (일간):**

- `dailyShowRelay` computed — 조건: `hasDailyScheduledTasks && !dailyUseDonut`
- `dailyRelayBody` computed, template `.task-planning__relay` 블록
- `pickDailyPraiseMessage`, `pickDailyAllClearMessage` import

**제거 (주간):**

- `weeklyShowRelay` computed
- `weeklyRelayBody` computed
- template 주간 `.task-planning__relay` 블록
- `pickWeeklyPraiseMessage`, `pickWeeklyAllClearMessage` import (task-feedback.ts에는 유지)

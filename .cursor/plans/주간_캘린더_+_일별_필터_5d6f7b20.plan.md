# 주간 일자 표기 + 업무 목록 일별 필터

## 날짜 규칙 (사용자 선택: C)

- 각 `Task`에 대해 **캘린더 일자 집합**을 계산한다.
  - `**startDate`와 `deadline` 둘 다 있음**: 로컬 날짜 기준 **닫힌 구간** `[start, end]`의 **모든 날에 일정이 포함된 것으로 간주. (`start`가 `end`보다 늦으면 스왑 또는 단일 날로 처리)
  - `**startDate`만: 그 하루만
  - `**deadline`만: 그 하루만
  - **둘 다 없음**: 주간 점/일별 필터 대상에서 제외

## UI 배치

- 위치: `[src/components/TaskList.vue](src/components/TaskList.vue)`에서 `**<div class="task-list__header">` 위 또는 헤더 바로 아래에 새 블록 추가 (예: `task-list__week`).
- **금주**: 현재 시각 기준 **월요일~일요일** (주 단위) 7칸. 각 칸에 요일 약글자 + 일자 숫자, 하단에 점 표시.
- **점 개수**: 해당 날짜에 일정이 걸린 **업무 수**(`task` 수) = 0이면 점 없음, 1이면 점 1개, **2 이상이면 점 2개** (최대 2개로 캡).
- **선택 상태**: 날짜 클릭 시 그날에 일정이 포함된 업무만 목록에 표시. **같은 날짜를 다시 클릭**하면 필터 해제(전체 목록)로 토글하는 UX 권장.
- **매트릭스**: 본 요청은 **업무 목록**만 필터. `[PriorityMatrix.vue](src/components/PriorityMatrix.vue)`는 전체 업무 유지(선택 시 동기화는 선택 사항, 범위 밖).

## 구현 포인트

1. **날짜 유틸** (신규 예: `src/utils/date-range.ts` 또는 기존 `[date-formatter](src/utils/date-formatter.ts)` 인접)

- ISO 문자열 → 로컬 `YYYY-MM-DD` 정규화
- `getWeekRange(anchor: Date): { start: Date; end: Date }` (월~일)
- `getTaskDatesOnCalendar(task): Set<string>` 또는 `getTaskSpanDays(task): string[]` (중복 제거, 날짜 순)

1. **상태**: `TaskList` 내부 `ref`로 `selectedCalendarDay: string | null` (`YYYY-MM-DD` 또는 `null` = 전체).
2. **표시용 데이터**:

- `taskStore.tasksWithComputed` 또는 `tasks`에서 `incompleteTasks` / `completedTasks` 계산 전에 **필터**: `selectedCalendarDay`가 있으면 `getTaskSpanDays(task)`에 해당 문자열이 포함되는 업무만.

1. **스타일**: SCSS 모듈/기존 `task-list` 스타일과 맞춤. 점은 작은 원 (`border-radius: 50%`), 선택된 날은 테두리/배경으로 강조.
2. **접근성**: 날짜는 `button` 또는 `role="button"` + `aria-pressed`/`aria-label` (예: “3월 30일, 일정 3건”).

## 테스트 (선택)

- Vitest: 유틸 함수에 대해 `start`/`deadline`만, 구간, 경계(같은 날), 타임존 없이 ISO 날짜만 검증하는 단위 테스트.
